# 우리 반 웹 프로그램 전시관

학생들이 AI와 함께 만든 웹 프로그램을 **미술관(갤러리) 형식**으로 전시하는 사이트입니다.
구글 시트를 그대로 읽어 오고, 방문자들이 **좋아요**와 **댓글**을 남길 수 있습니다.

---

## 이 사이트가 하는 일

- 구글 시트에서 `이름 · 앱제목 · 앱설명 · 사이트주소 · 선생님평가`만 읽어 전시
- **선생님 평가 별 0개인 앱은 자동 제외**, 나머지는 **선생님 평가 높은 순**으로 배치
- 이름 가운데 글자는 `*`로 가림 (개인정보 보호), 학년·반은 표시하지 않음
- 화면의 **별은 방문자 좋아요 수**로 채워짐 (좋아요 2개당 별 1개, 최대 5개, 0부터 시작)
- 한 사람(브라우저)이 같은 앱에 좋아요는 **한 번만**
- 사이트 주소가 없는 앱은 방문 버튼을 숨김

---

## 1. 바로 실행해 보기 (미리보기)

폴더에서 아래 명령을 실행한 뒤 브라우저로 `http://localhost:8777` 접속:

```bash
python3 -m http.server 8777
```

이 상태(=Firebase 미설정)는 **미리보기 모드**입니다.
좋아요·댓글이 "그 브라우저에만" 저장되고 다른 사람과 공유되지 않습니다.
공유하려면 아래 2번을 설정하세요.

---

## 2. 좋아요·댓글을 모두에게 공유하기 (Firebase 설정)

무료로 가능합니다. 약 10분.

1. https://console.firebase.google.com → **프로젝트 만들기**
2. 왼쪽 **빌드 → Firestore Database → 데이터베이스 만들기** (위치는 asia-northeast3(서울) 권장)
3. 프로젝트 설정(⚙️) → **내 앱 → 웹(&lt;/&gt;) 앱 추가** → 나오는 `firebaseConfig` 값을 복사
4. `js/config.js` 의 `firebase: { ... }` 안에 그 값(apiKey, projectId 등)을 붙여넣기
   - `apiKey`는 비밀이 아니므로 그대로 넣어도 안전합니다.
5. **보안 규칙 적용** (중요): Firestore → **규칙(Rules)** 탭을 열고,
   이 폴더의 `firestore.rules` 내용을 그대로 붙여넣은 뒤 **게시**.
   → 좋아요는 +1씩만, 댓글은 길이 제한이 걸려 장난·조작을 막습니다.

설정 후 새로고침하면 하단 문구가 "실시간으로 공유됩니다"로 바뀝니다.

---

## 3. 인터넷에 올리기 (배포) — 택1

정적 사이트라 어디든 올라갑니다.

- **Firebase Hosting** (Firebase를 이미 쓰면 자연스러움)
  ```bash
  npm i -g firebase-tools
  firebase login
  firebase init hosting   # public 폴더로 이 폴더 지정
  firebase deploy
  ```
- **GitHub Pages**: 이 폴더를 깃허브 저장소에 올리고 Settings → Pages → 브랜치 선택
- **Netlify / Vercel**: 폴더를 드래그&드롭 하거나 깃 저장소 연결

---

## 4. 구글 시트가 바뀌면 (업데이트 방법)

- **내용만 수정**(설명·별점·주소 등): 시트에서 고치면 사이트 **새로고침만으로 자동 반영**됩니다. 할 일 없음.
- **시트 링크(주소)가 아예 바뀐 경우**: `js/config.js` 의 `SHEET_ID` 한 줄만 새 ID로 교체.
  - ID는 시트 주소에서 `/d/` 와 `/edit` 사이의 긴 문자열입니다.
- 시트는 반드시 **"링크가 있는 모든 사용자 — 뷰어"** 공개로 두어야 읽힙니다.
- 소개글을 손보고 싶으면 `js/config.js` 의 `DESCRIPTION_OVERRIDES` 에서 편집.

---

## 5. 실제 앱 스크린샷 넣기

기본은 앱 제목·아이콘 썸네일입니다. 실제 화면으로 바꾸려면
`images/` 폴더에 `앱제목.png` 파일을 넣으면 자동 교체됩니다. (자세한 건 `images/README.md`)

> 참고: 학생 링크의 상당수가 Gemini "대화 공유" 링크라, 제작자 로그인 없이는
> 열리지 않습니다(방문자에게 "링크가 존재하지 않음"). 가능하면 학생들에게
> **배포된 앱 주소**(예: `*.lovable.app`)나 **직접 찍은 스크린샷**을 받아 두는 걸 권장합니다.

---

## 6. 추가로 넣으면 좋은 기능 (제안)

- **QR 코드**: 전시회장에서 각 앱 QR을 찍어 바로 접속 (스크린샷 대체로도 유용)
- **검색·필터**: 제목/제작자 검색, 카테고리(게임·학습·도구) 필터
- **정렬 토글**: "선생님 평가순 ↔ 좋아요순" 전환 버튼
- **관리자 모드**: 부적절한 댓글 숨김/삭제 (Firebase Auth로 선생님만)
- **조회수**: 각 앱이 몇 번 열렸는지 집계
- **주간 인기작**: 이번 주 좋아요가 많은 작품 상단 배너
- **다운로드/공유**: 전시 페이지 자체를 SNS로 공유하는 버튼
- **접근성**: 키보드 탐색·스크린리더 라벨 보강

## 파일 구조

```
index.html          화면 틀
css/style.css        갤러리(미술관) 디자인
js/config.js         ⚙️ 설정 (시트 ID, Firebase, 소개글) — 여기만 고치면 됨
js/app.js            로직 (시트 읽기·정렬·좋아요·댓글)
firestore.rules      Firebase 보안 규칙 (좋아요/댓글 악용 방지)
images/              실제 스크린샷 넣는 곳
```

## 알려진 한계

- 좋아요 중복 방지는 **브라우저 기준**입니다(로그인 없음). 다른 기기·시크릿창에서는
  다시 누를 수 있습니다. 더 엄격히 막으려면 Firebase 익명 로그인 + App Check 추가가 필요합니다.
